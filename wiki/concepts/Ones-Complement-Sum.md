# One's Complement Sum (1의 보수 덧셈)

한 줄 요약: 오버플로우 carry를 버리지 않고 하위 비트에 다시 더하는 덧셈. 네트워크 checksum의 기반.

## 직관

일반 덧셈에서 자리올림(carry)은 버린다. 1의 보수 덧셈은 버리는 대신 **끝에서 다시 더한다** (end-around carry). 덧셈 결과가 항상 정해진 비트 수 안에 머무른다.

시계 덧셈과 비슷하다: 12 + 3 = 3 (12를 넘으면 처음으로 돌아옴). 1의 보수 덧셈도 비트 범위를 순환한다.

## 동작 방식

1. 16비트 단위로 데이터 나누기
2. 더하기 (일반 이진 덧셈)
3. **캐리 발생 시**: 캐리를 결과 하위 비트에 다시 더함 (end-around carry)
4. 최종 결과에 **1의 보수 취하기** (비트 반전) → Checksum 완성

## 예시

```
  1111 0000 0000 0001
+ 0001 0000 0000 0001
= 1 0000 0000 0000 0010   ← 17번째 비트에 캐리 발생

end-around carry: 캐리(1)를 하위 16비트에 더함
  0000 0000 0000 0010
+                    1
= 0000 0000 0000 0011   ← 최종 합

checksum = 1의 보수 = 1111 1111 1111 1100
```

## 검증 원리

수신 측이 같은 방식으로 재계산하면:
- 정상: 모든 비트가 1 → 1의 보수 취하면 **0**
- 오류: 0이 아님 → 패킷 손상됨

## 특징

- **교환법칙 성립**: 더하는 순서 무관하게 동일 결과
- **하드웨어 구현 단순**
- **오류 검출용** (수정 기능 없음)

## 사용 사례

| 프로토콜 | 적용 범위 |
|---------|---------|
| UDP | Checksum 계산 |
| TCP | Checksum 계산 |
| IP | Header Checksum |

## 왜 이렇게 설계됐는가

2의 보수 덧셈은 하드웨어 연산이 빠르지만, 결과 검증이 복잡하다. 1의 보수는 검증이 단순하다: 보내고 받아서 더하면 전부 1. 네트워크 프로토콜이 설계될 당시(1970s) 하드웨어 제약에서 최적의 균형점.
