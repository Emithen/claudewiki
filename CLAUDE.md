- 연관 개념을 동반해서 이해할 때 빠르고 단단하게 습득

### 빌더
- 웹 소프트웨어 개발
- 개념을 실험할 수 있는 샌드박스/데모 키트 제작

### 지식 전달자
- 내가 공부한 주제에 다른 사람을 끌어들이기 위한 가이드 문서·영상 제작
- 전공 개념을 이해하기 쉬운 문서로 정리

## AI에게 기대하는 것

- **기록 비용을 낮춰줘**: 내가 생각하는 속도를 기록이 못 따라가는 게 핵심 문제. 초안을 빠르게 만들어주고, 내가 다듬는 구조로.
- **구조적 유사성·비유를 활용해서 설명해줘**: 다른 도메인의 유사한 구조를 연결해주면 이해가 빠름.
- **"왜 이렇게 설계됐는가"를 함께 짚어줘**: 정의 나열보다 설계 선택지와 trade-off가 더 유용함.
- **재귀적 탐색을 적절히 조절해줘**: 초반부터 너무 깊이 파고들지 않도록 학습 흐름을 잡아줘.

## 볼트 구조

```
raw/          불변 원본. AI는 읽기만 함.
  lectures/   수업 자료, 강의 노트
  books/      책 메모, 독서 노트
  articles/   웹 아티클, 스크랩
  videos/     영상 노트
  thoughts/   흘러가는 생각, 아이디어

wiki/         AI가 컴파일하고 유지하는 지식베이스.
  concepts/   개념 문서 (한 파일 = 한 개념)
  connections/ 개념 간 연결, 구조적 유사성, 비교
  index.md    전체 페이지 목록 (AI가 갱신)
  log.md      작업 기록 (append-only)

output/       wiki에서 파생된 외부용 결과물.
  guides/     가이드 문서, 튜토리얼
  demos/      샌드박스 코드
  posts/      블로그·SNS용 글
```

## 위키 운영 워크플로우

### Ingest (새 소스 추가)
1. soggyfries가 `raw/` 에 파일 추가 후 "인제스트해줘" 요청
2. AI: 소스 읽기 → 핵심 포인트 논의 → wiki에 반영
   - `wiki/concepts/` 관련 페이지 생성 또는 업데이트
   - `wiki/connections/` 구조적 유사성 있으면 연결 문서 작성
   - `wiki/index.md` 갱신
   - `wiki/log.md` 기록 추가

### Query (질문)
- AI가 index.md → 관련 페이지 순으로 탐색 후 답변
- 좋은 답변은 wiki 페이지로 역파일 가능 (soggyfries가 판단)

### Lint (위키 점검)
- "위키 점검해줘" 요청 시 AI가 수행:
  - 모순·충돌 페이지 탐색
  - 고아 페이지 (링크 없는 페이지) 탐색
  - 개념은 언급됐지만 페이지 없는 항목 탐색
  - 보완할 소스 제안
