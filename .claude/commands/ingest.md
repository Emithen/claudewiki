raw/ 폴더에서 새로 추가된 파일을 찾아 wiki에 통합하는 전체 프로세스를 실행해줘.

---

## Step 1: 새 파일 탐지

아래 두 가지 범주를 모두 스캔해줘.

### 범주 A — 웹 클리퍼 파일 (하위 폴더)
`raw/articles/`, `raw/videos/`, `raw/docs/` 안의 파일 중
frontmatter의 `status`가 `unread` 또는 `unwatched`인 것.

### 범주 B — 직접 작성 파일 (raw/ 루트)
`raw/` 루트에 직접 있는 파일 (`CLAUDE.md` 제외).
이 파일들은 frontmatter가 없을 수 있어. wiki/sources/나 wiki/concepts/에 아직 반영되지 않은 것으로 간주해.
이미 wiki에 반영된 것은 wiki/log.md에서 확인해서 제외해줘.

### 탐지 결과 출력
두 범주를 구분해서 표로 정리:

| 파일명 | 경로 | 범주 | 타입 추정 | 날짜 |
|--------|------|------|----------|------|

파일이 여러 개면 사용자에게 어떤 파일을 인제스트할지 선택하게 해줘.
파일이 하나면 바로 진행해줘.

---

## Step 2: 파일 읽기

선택한 파일을 읽어줘. **raw/ 파일은 절대 수정하지 않는다.**

---

## Step 3: 소스 요약

파일 내용을 분석해서 다음을 정리해줘:

**Key Claims**: 이 소스의 핵심 주장 또는 핵심 정보 (불릿 3~7개)

**Entities**: 언급된 사람, 도구, 프레임워크, 조직 (표로)

**Concepts**: 다루는 개념 목록 (wiki 페이지로 만들 후보)

---

## Step 4: 사용자 인터뷰

요약을 보여준 뒤, 다음 3가지를 **한 번에** 질문해줘:

1. 이 소스를 왜 캡처했어?
2. 지금 하고 있는 일과 어떻게 연결돼?
3. 이걸로 뭘 해보고 싶어? (wiki 개념 정리 / output 가이드 / 샌드박스 코드 / 기타)

---

## Step 5: wiki 반영

사용자 답변을 받으면 다음 순서로 wiki를 업데이트해줘.

### 5-1. 소스 요약 페이지 생성
경로: `wiki/sources/YYYY-MM-DD_소스제목.md`

frontmatter 포함:
```
---
title:
type:
source:
product:
clipped:
status: ingested
tags:
---
```

내용: Key Claims, Entities, Concepts, 관련 wiki 페이지 링크, 생성된 아웃풋 링크

### 5-2. 개념 페이지 생성 또는 업데이트
경로: `wiki/concepts/개념명.md`

각 개념 페이지는 반드시 아래 구조를 따라줘:
```
# 개념명

한 줄 요약: 이 개념이 왜 존재하는가.

## 직관
코드 없이 말로. 다른 도메인의 구조적 유사성이 있으면 반드시 연결.

## 개념 설명
핵심 내용을 구조적으로.

## 코드 예제
최대한 단순하게, 직접 실행 가능하게.

## 왜 이렇게 설계됐는가
설계 선택지와 trade-off.

## 연결된 개념
- [[관련 개념 1]]
- [[관련 개념 2]]
```

이미 존재하는 개념 페이지가 있으면 새 소스의 내용을 통합해서 업데이트해줘.
기존 내용과 충돌하면 해당 섹션에 `> ⚠️ 충돌:` 블록으로 표시하고 사용자에게 알려줘.

### 5-3. 연결 문서 생성 (해당하는 경우)
경로: `wiki/connections/개념A-vs-개념B.md` 또는 `wiki/connections/개념A-상위추상화.md`

이 소스에서 구조적 유사성 또는 흥미로운 개념 간 연결이 발견되면 connections/ 에 문서를 만들어줘.
양쪽 개념 페이지에 역링크도 추가해줘.

### 5-4. 아웃풋 생성 (사용자가 원하는 경우)
사용자가 가이드, 샌드박스 코드, 포스트 등을 요청했으면:

- 가이드: `output/guides/제목.md`
- 샌드박스: `output/demos/제목/`
- 포스트: `output/posts/제목.md`

아웃풋은 초안으로 만들어도 됨. 빠른 게 중요.

---

## Step 6: index.md, log.md 업데이트

### wiki/index.md 업데이트
- Sources, Concepts, Connections 각 섹션에 새 페이지 추가
- 마지막 줄의 날짜와 총 페이지 수 갱신

### wiki/log.md 업데이트
다음 형식으로 맨 위에 항목 추가:
```
## [YYYY-MM-DD] ingest | 소스 제목

- 소스: raw/경로
- 생성: wiki/sources/...
- 생성 또는 업데이트: wiki/concepts/...
- 생성: output/...  (해당하는 경우)
- 컨텍스트: 사용자가 말한 연결 맥락 한 줄 요약
```

---

## 주의사항

- raw/ 파일은 절대 수정하지 않는다.
- 개념 설명 시 구조적 유사성과 비유를 적극 활용한다 (사용자의 학습 방식).
- "왜 이렇게 설계됐는가"를 항상 짚는다.
- 초안이어도 괜찮다. 빠르게 만드는 것이 우선.
