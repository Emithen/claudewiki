wiki/ 전체를 점검해서 깨지거나 최신화가 안 된 것들을 찾아 정리해줘.

---

## Step 1: 전체 파일 수집

wiki/ 하위의 모든 .md 파일 목록을 수집해줘 (index.md, log.md 포함).

---

## Step 2: 점검 항목별 검사

아래 항목을 순서대로 점검해줘.

### 2-1. 고아 페이지 (Orphan)
다른 페이지에서 링크되지 않은 페이지를 찾아줘.
index.md에도 없고 어떤 페이지의 `## 연결된 개념`에도 없는 페이지.

### 2-2. 깨진 링크 (Broken Link)
`[[페이지명]]` 형식으로 링크됐지만 실제 파일이 존재하지 않는 것들.
모든 페이지에서 `[[...]]` 패턴을 수집해서 실제 파일과 대조해줘.

### 2-3. 개념은 있지만 페이지 없음 (Missing Page)
여러 페이지에서 언급되거나 링크됐지만 wiki/concepts/에 파일이 없는 개념들.
새 소스가 인제스트될 때 생긴 링크인데 페이지가 만들어지지 않은 경우.

### 2-4. index.md 불일치
wiki/ 폴더에 실제로 존재하는 파일인데 index.md에 없는 것.
반대로 index.md에는 있는데 실제 파일이 없는 것.

### 2-5. 오래된 소스 (Stale)
wiki/sources/ 페이지 중 `status: ingested`인데 그 이후 연결된 개념 페이지가 없는 것.
인제스트됐지만 사실상 아무것도 연결 안 된 소스.

### 2-6. 구조 미완성 페이지 (Incomplete)
wiki/concepts/ 페이지 중 아래 섹션이 빠진 것:
- `## 직관`
- `## 왜 이렇게 설계됐는가`
- `## 연결된 개념`

---

## Step 3: 점검 결과 리포트

다음 형식으로 리포트를 출력해줘:

```
## Lint 결과 — YYYY-MM-DD

### 고아 페이지 (N개)
- ...

### 깨진 링크 (N개)
- [파일명] → [[링크]] 존재하지 않음

### 개념 페이지 미생성 (N개)
- [[개념명]] — N개 페이지에서 참조됨

### index.md 불일치 (N개)
- 파일 있음, index 없음: ...
- index 있음, 파일 없음: ...

### 연결 없는 소스 (N개)
- ...

### 구조 미완성 페이지 (N개)
- [파일명] — 빠진 섹션: ...

---
이상 없음 항목: ...
```

---

## Step 4: 수정 실행

리포트를 보여준 뒤 사용자에게 물어봐줘:

> "위 항목들을 자동으로 수정할까요?
> 전체 수정 / 항목별 선택 / 리포트만 보기"

사용자가 수정을 선택하면 아래 기준으로 처리해줘:

| 항목 | 자동 처리 방법 |
|------|--------------|
| 고아 페이지 | index.md에 추가 + 관련 페이지에 역링크 추가 제안 |
| 깨진 링크 | 링크 삭제 또는 대상 페이지 생성 (사용자 선택) |
| 개념 페이지 미생성 | 기존 링크 언급 내용 기반으로 stub 페이지 자동 생성 |
| index.md 불일치 | index.md에 추가하거나 죽은 항목 제거 |
| 연결 없는 소스 | 소스 내용 재분석 후 concepts/ 연결 추가 |
| 구조 미완성 | 빠진 섹션을 채워줌 (내용 기반으로 초안 생성) |

---

## Step 5: log.md 업데이트

수정 완료 후 log.md 맨 위에 항목 추가:

```
## [YYYY-MM-DD] lint | Wiki 점검

- 고아 페이지: N개 발견 / N개 수정
- 깨진 링크: N개 발견 / N개 수정
- 개념 페이지 미생성: N개 발견 / N개 생성
- index.md 불일치: N개 수정
- 구조 미완성: N개 수정
- 보완 추천 소스: (있으면 목록)
```

---

## 주의사항

- raw/ 파일은 절대 수정하지 않는다.
- 자동 수정 시 기존 내용을 덮어쓰지 않는다. 추가와 보완만 한다.
- 확신이 없는 수정은 먼저 사용자에게 확인한다.
